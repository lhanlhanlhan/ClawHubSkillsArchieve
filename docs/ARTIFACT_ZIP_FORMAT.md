# Artifact ZIP Directory Format

This document defines the standard directory layout of the ZIP file generated by GitHub Actions artifacts.

## ZIP Root

The artifact ZIP contains a single top-level folder:

```
archives/
```

## Standard Layout

```
archives/
  top{N}/
    metadata/
      {owner}__{slug}__meta.json
    skills/
      {owner}/
        {slug}/
          SKILL.md
          _meta.json
          [other files...]
    top{N}_skills_manifest.json
    top{N}_skills_summary.csv
```

## Field Notes

- `{N}` is the requested top count (e.g., 50, 100).
- `{owner}` is the GitHub owner name.
- `{slug}` is the skill slug path used in the source repository.
- `metadata/` stores raw `_meta.json` for each archived skill.
- `skills/` stores the full skill directory snapshot.
- `top{N}_skills_manifest.json` is the complete JSON manifest for the run.
- `top{N}_skills_summary.csv` is a compact CSV summary for quick inspection.

## Artifact Naming

The artifact name follows this pattern:

```
clawhub-top-{N}-{run_id}
```

## Example

```
archives/
  top50/
    metadata/
      steipete__gog__meta.json
    skills/
      steipete/
        gog/
          SKILL.md
          _meta.json
          CONTRIBUTING.md
    top50_skills_manifest.json
    top50_skills_summary.csv
```
